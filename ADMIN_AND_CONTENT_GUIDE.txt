AimAze Website Admin / Content Control Guide

1) Public admin panel
/admin.html is intentionally disabled for security. The earlier static admin panel exposed a default password in frontend code, which is not acceptable for an enterprise website. Do not publish a frontend-only admin panel with a visible password.

2) Current safe way to update website content
Edit content.json before uploading/deploying:
- site.phone: public phone number, e.g. +971 XX XXX XXXX
- site.whatsapp: WhatsApp number without + or spaces, e.g. 9715XXXXXXXX
- hero: homepage headline, text, CTA labels and links
- stats: homepage metric cards
- services: service cards
- nav.links: menu items

3) Files normally edited for content
- content.json: main business content and contact details
- index.html: homepage layout only
- site.css: design styling for inner pages
- shared.js: common navbar/footer/WhatsApp/content behavior

4) Lead tools
- Floating WhatsApp button reads the number from content.json > site.whatsapp.
- Lead tracking is handled by lead-capture.js in browser storage for the demo/static version.
- lead-dashboard.html is a local/static lead dashboard. For production, connect this to Odoo CRM, Firebase, Supabase, Netlify Forms, or a protected backend database.

5) Recommended enterprise admin options
Best option for AimAze: connect website form directly to Odoo CRM and manage leads inside Odoo.
Other options: WordPress CMS, Decap/Netlify CMS with GitHub login, Firebase/Supabase admin dashboard, or a custom Node/Python backend with real login authentication.
